Modern C# type system · Module 01

Modeling with modern C# types.

A practical module for backend developers who want to use C# types to express identity, value, equality, stable keys, small domain concepts, and valid state transitions.

Open source module Repository

Learning objectives

What you should know after this module

  • Explain the difference between identity and value.
  • Choose between class, record class, struct, record struct, and readonly record struct.
  • Predict how generated equality affects HashSet<T> and Dictionary<TKey, TValue>.
  • Replace ambiguous primitive IDs with strongly typed IDs.
  • Model valid states with explicit types instead of boolean flags and nullable fields.

Core idea

A good type makes the valid path natural.

Modern C# features are not only syntax shortcuts. Used deliberately, they communicate whether a concept is a long-lived entity, a value equal by content, a stable key, or a state that carries only the data valid for that state.

Module path

The progression

  1. Identity vs value.
  2. Records and generated equality.
  3. Stable keys for hash-based collections.
  4. Strongly typed IDs.
  5. Value objects.
  6. record struct mutability vs readonly record struct.
  7. Type-driven state modeling.
  8. Serialization choices for record hierarchies.

Exercises

Practice prompts

  • Replace a method with several Guid parameters by strongly typed IDs.
  • Break a dictionary lookup with a mutable key, then fix it with a stable value key.
  • Model an onboarding workflow with explicit states instead of flags and nullable fields.
