# Site direction

The public site should feel like a technical documentation system with a portfolio-style landing page.

The projects are architecture-heavy and decision-heavy. The site should make those decisions easy to find without making the homepage feel like a generic docs portal.

## Principles

- Keep the main site global.
- Keep project documentation with project repositories.
- Link to ADRs where they matter.
- Treat learning paths as first-class content, not only sample code.
- Avoid copying generated docs into this repository.
