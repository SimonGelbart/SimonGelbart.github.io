# Design system

The design should be calm, technical, and documentation-friendly.

The main site and project docs should share enough visual language that they feel like one ecosystem, even when the source content lives in different repositories.

## Direction

- Warm neutral background.
- Green accent inherited from the existing static site.
- Large editorial homepage headings.
- Card-based project and learning-path indexes.
- Starlight documentation pages for durable publishing guidance.

## Future components

- Project cards.
- ADR status badges.
- Course lesson callouts.
- Prediction question blocks.
- Rule-of-thumb blocks.
- Optional lab cards.
